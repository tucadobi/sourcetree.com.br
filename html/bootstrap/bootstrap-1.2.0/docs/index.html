Bootstrap

Bootstrap é um kit de ferramentas projetado para iniciar o desenvolvimento de aplicativos e sites da web.
Inclui CSS e HTML base para tipografia, formulários, botões, tabelas, grids, navegação e muito mais.

Observação: Bootstrap é construido com Less e foi projetado para funcionar desde o início com os navegadores modernos em mente.

Faça um link direto para o CSS

Para um início mais rápido e fácil, basta copiar este snippet em sua página da web. 

<link rel="stylesheet" type="text/css" href="http://getbootstrap.com.br/1.2.0/assets/css/bootstrap-1.2.0.css">
Use-o com Less

Fã de usar Less ? Não tem problema, basta clonar o repositório e adicionar estas linhas: 

<link rel="stylesheet/less" href="/path/to/bootstrap.less">
<script type="text/javascript" src="/path/to/less.js"></script>

Sobre o Bootstrap Breve histórico, suporte ao navegador e muito mais

História

Nos primeiros dias, os engenheiros usavam quase todas as bibliotecas com as quais estavam familiarizados para atender aos requisitos de front-end. Bootstrap começou como uma resposta aos desafios apresentados e o desenvolvimento acelerou rapidamente durante a primeira Hackweek.

Com a ajuda e feedback de muitos engenheiros, o Bootstrap cresceu significativamente para abranger não apenas estilos básicos, mas também padrões de design de front-end mais elegantes e duráveis.

Misc. elementos

Usando ênfase, endereços e & abreviaturas

<strong> <em> <address> <abbr>

Quando usar

Tags de ênfase (<strong> e <em>) deve ser usado para indicar importância ou ênfase adicional de uma palavra ou frase em relação ao texto ao seu redor. Use <strong> para importância e <em> para ênfase de stress.

Ênfase em um parágrafo

Fique tranquila, me ame, hoje, ontem, desejos, sofridos por você, você, minha, vida, meu tudo, adeus. Oh, continue a me amar, jamais duvide do coração fiel de seu amado. Fique tranquila, me ame, hoje.

Observação: Ainda não há problema em usar tags <b> e <i> em HTML5 e não precisam ser estilizados em negrito e itálico, respectivamente (embora se houver um elemento mais semântico, use-o). <b> destina-se a destacar palavras ou frases sem transmitir importância adicional, enquanto <i> é principalmente para voz, termos técnicos, etc.

Endereços

O elemento <address> é usado para informações de contato de seu ancestral mais próximo ou de todo o corpo da obra. Veja como parece:

bWrite, Inc.
São Paulo, São Paulo Ap 103
Rua dos pinheirinhos
Telefone: (11) 9 8002-2829

Observação: Cada linha em um <address> e deve terminar com uma quebra de linha (<br>) ou ser encapsulado em uma tag em nível de bloco (por exemplo, <p>) estruturar adequadamente o conteúdo.

Abreviações

Para abreviações e siglas, use o <abbr> tag (<acronym> está obsoleto no HTML5). Coloque o formulário abreviado dentro da tag e defina um título para o nome completo.

Blockquotes

<blockquote> <p> <small>

Como citar

Para incluir uma citação em bloco, envolva <blockquote> em volta das tags <p> e <small>. Use o elemento <small> para citar sua fonte e você receberá um travessão &mdash; antes disso.

Tabelas Pois, você adivinhou, dados tabulares

Construindo tabelas

<table> <thead> <tbody> <tr> <th> <td> <colspan> <caption>

As tabelas são ótimas para muitas coisas. Grandes tabelas, no entanto, precisam de um pouco de marcação para serem úteis, escalonáveis e legíveis (no nível do código). Aqui estão algumas dicas para ajudar.

Sempre coloque os títulos das colunas em um <thead> tal que a hierarquia seja <thead> > <tr> > <th>.

Semelhante aos cabeçalhos das colunas, todo o conteúdo do corpo da sua tabela deve ser agrupado em um <tbody> então sua hierarquia é <tbody> > <tr> > <td>.

Exemplo: estilos de tabela padrão

Todas as tabelas serão estilizadas automaticamente apenas com as bordas essenciais para garantir a legibilidade e manter a estrutura. Não há necessidade de adicionar classes ou atributos extras.

2 Samara Tamiris Português
3 Amanda Paula Português

Botões

Por convenção, os botões são usados para ações enquanto os links são usados para objetos. Por exemplo, “Download” pode ser um botão e “atividade recente” pode ser um link.

Todos os botões têm como padrão um estilo cinza claro, mas diversas classes funcionais podem ser aplicadas para diferentes estilos de cores. Essas classes incluem uma class azul .primary, uma class azul claro .info, uma class verde .success, e a class vermelho .danger. Além disso, criar seus próprios estilos é fácil.

O que é

Nossa barra superior é uma barra fixa que abriga o logotipo ou nome de um site, a navegação principal e o formulário de pesquisa.

Customizável

Todos os elementos contidos nele, e também toda a barra superior, são opcionais. Você pode optar por incluir um logotipo/nome, navegação, pesquisa e uma navegação secundária - ou qualquer combinação disso.

Dropdowns incluído

Como parte da navegação principal, incluímos a capacidade de adicionar menus suspensos à sua navegação. Confira a navegação secundária acima (alinhada à direita) para ver como isso é feito.

Observação: Ao usar a barra superior em qualquer página, leve em consideração a sobreposição que ela causa adicionando padding-top: 40px; para o seu body.

Usando Bootstrap com Less Incremente seu CSS com variáveis, mixins e funções

Bootstrap foi construído com Preboot, um pacote de mixins e variáveis de código aberto para ser usado em conjunto com Less, um pré-processador CSS para desenvolvimento web mais rápido e fácil.

Confira como usamos o Preboot no Bootstrap e como você pode usá-lo caso opte por executar o Less em seu próximo projeto.

Como usá-lo

Use esta opção para fazer uso completo de variáveis Less, mixins e aninhamento em CSS do Bootstrap via javascript em seu navegador. 

<link rel="stylesheet/less" href="less/bootstrap.less" media="all" />
<script type="text/javascript" src="js/less-1.1.3.js"></script>

Não está sentindo a solução .js ? Experimente o aplicativo Less para Mac ou use Node.js para compilar ao implantar seu código.

O que está incluído

Aqui estão alguns destaques do que está incluído no Bootstrap como parte do Bootstrap. Acesse o site Bootstrap ou a página do projeto Git para baixar e saber mais.

Variáveis

Variáveis em Less são perfeitas para manter e atualizar seu CSS sem dor de cabeça. Quando você quiser alterar um valor de cor ou um valor usado com frequência, atualize-o em um ponto e pronto. 

// Links
@linkColor: #8b59c2;
@linkColorHover: darken(@linkColor, 10);

// Grays
@black: #000;
@grayDark: lighten(@black, 25%);
@gray: lighten(@black, 50%);
@grayLight: lighten(@black, 70%);
@grayLighter: lighten(@black, 90%);
@white: #fff;

// Accent Colors
@blue: #08b5fb;
@green: #46a546;
@red: #9d261d;
@yellow: #ffc40d;
@orange: #f89406;
@pink: #c3325f;
@purple: #7a43b6;

// Baseline grid
@basefont: 13px;
@baseline: 18px;

Comentando

Less também fornece outro estilo de comentários além da sintaxe /* ... */ normal do CSS. 

// Este é um comentário.
/* Este também é um comentário. */

Mistura o wazoo

Mixins são basicamente inclusões ou parciais para CSS, permitindo combinar um bloco de código em um. Eles são ótimos para propriedades prefixadas de fornecedores, como box-shadow, gradientes entre navegadores, pilhas de fontes e muito mais. Abaixo está um exemplo dos mixins incluídos no Bootstrap.

Pilhas de fontes 

#font {
    .shorthand(@weight: normal, @size: 14px, @lineHeight: 20px)
    {
        font-size: @size;
        font-weight: @weight;
        line-height: @lineHeight;
    }

    .sans-serif(@weight: normal, @size: 14px, @lineHeight: 20px)
    {
        font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
        font-size: @size;
        font-weight: @weight;
        line-height: @lineHeight;
    }

    .serif(@weight: normal, @size: 14px, @lineHeight: 20px)
    {
        font-family: "Georgia", Times New Roman, Times, sans-serif;
        font-size: @size;
        font-weight: @weight;
        line-height: @lineHeight;
    }

    .monospace(@weight: normal, @size: 12px, @lineHeight: 20px)
    {
        font-family: "Monaco", Courier New, monospace;
        font-size: @size;
        font-weight: @weight;
        line-height: @lineHeight;
    }
}

Operações e sistema de grid

Seja sofisticado e faça algumas contas para gerar mixins flexíveis e poderosos como o mostrado abaixo. 

// Griditude
@gridColumns: 16;
@gridColumnWidth: 40px;
@gridGutterWidth: 20px;
@siteWidth: (@gridColumns * @gridColumnWidth) + (@gridGutterWidth * (@gridColumns - 1));

// Grid System
.container
{
    width: @siteWidth;
    margin: 0 auto;

    .clearfix();
}

.columns(@columnSpan: 1)
{
    width: (@gridColumnWidth * @columnSpan) + (@gridGutterWidth * (@columnSpan - 1));
}

.offset(@columnOffset: 1)
{
    margin-left: (@gridColumnWidth * @columnOffset) + (@gridGutterWidth * (@columnOffset - 1)) + @extraSpace;
}
